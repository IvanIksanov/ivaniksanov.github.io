Задания на собеседованиях для QA

Задание 1. Победитель в Flappy Bird

Необходимо поменять количество очков в игре Flappy Bird. Воспользуйся DevTools, чтобы счетчик отобразил счет: 999!

Шаги по воспроизведению:

1. Открыть Devtools
2. Вкладка Source
3. Открыть файл script.js
...

Задание 2. Практика Swagger

Проверить ответ на API запрос после нажатия на кнопку "Отправить".

Шаги по воспроизведению:

1. Открыть Devtools
2. Вкладка Network
3. Очистить сетевой журнал
4. Заполнить необходимые поля
5. Нажать кнопку "Отправить"
6. Найти API запрос в сетевом журнале
7. Отобразить значение во вкладке "Respones"

Задание 3. Удали лишний элемент

Удали со страницы часть заголовка "ИГРА" формата .svg

Шаги по воспроизведению:

1. Открыть Devtools
2. Вкладка Elements
3. Нажать на "Цель"
4. Переместить курсор на необходимый элемент
5. В структуре html удали элемент

Задание 4. Поставь Тетрис на паузу

Останови игру с помощью точки останова (breakpoint).
Поставить breakpoint на шаг кода, чтобы поставить игру на паузу.

Шаги по воспроизведению:

1. Открыть Devtools
2. Вкладка Source
3. Открыть файл script.js
4. Переместить курсор на строчку 362
5. Поставить точку останова нажав на "362"
6. Дождитесь падения текущей фигуры или положите её самостоятельно
7. Воспроизведи игру с помощью области управления breakpoint (см. Скрин)
НЕ ЗАБУДЬ УБРАТЬ точку останова после проверки :)

Задание 5. Открыть сайт в разных браузерах (Тестовое задание)

Проверь верстку страницы на разных клиентах.
Важно сравнивать верстку не только на разных браузерах, но и на разных движках.
Не забудь про размеры экранов.

Шаги по воспроизведению:

1. Открыть сайт в Chrome (движок Blink)
2. Открыть сайт в Safari (движок WebKit)
3. Открыть сайт в Mozilla Firefox (движок Gecko)
4. Открыть сайт в Internet Explorer (движок Trident)
5. Сравнить отрисовку элементов на странице
6. Проверить верстку - размер, стиль и расположение элементов
